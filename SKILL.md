# 小学阅读答题全面性评估与班级学情分析

## 概述

此技能使用AI技术实现学生阅读答题的自动化评估与班级学情分析。

**核心功能**：
- 逐人评估：给出明确的✅正确/⚠️基本正确/❌错误评估结果
- 要点覆盖分析：逐要点统计覆盖情况（完整/部分/缺失）
- 错误原因诊断：分析全班集中错误的具体原因
- 改进方案生成：针对薄弱点提供可落地的教学优化建议

**适用场景**：
- 小学1-6年级语文阅读理解作业批改
- 班级学情分析
- 批量评估学生作答图片（1-50张）

---

## 执行工作流程

### 步骤1：确认输入信息

收集以下必填信息：
- 学生作答图片（PNG/JPG/JPEG，1-50张/批次）
- 年级（1-6年级）
- 题目内容
- 参考答案要点（至少2个要点）

可选信息：
- 阅读文本（原文片段）
- 班级标识（如"三年级(2)班"）

如信息不足，根据图片内容自动推断年级和题目，根据常见阅读理解题型生成合理的参考答案要点，继续执行而不暂停询问用户。

### 步骤2：读取所有图片

1. 使用 `execute_command` 查找所有图片文件
2. 使用 `read_file` 工具并行读取所有图片
3. 必须读取完所有图片才能继续下一步
4. 如图片读取失败，记录错误但继续读取其他图片

**关键**：读取图片后，立即进入步骤3，不停顿。

### 步骤3：逐人评估（核心步骤）

对每张图片执行评估：

1. 识别学生作答内容（OCR）
2. 提取关键词和要点
3. 与参考答案要点对比
4. 计算要点覆盖情况：
   - ✅ 完整覆盖：学生回答完整、准确包含该要点的核心含义（得1分）
   - ⚠️ 部分覆盖：学生回答涉及该要点，但表述不完整或部分错误（得0.5分）
   - ❌ 缺失：学生回答未涉及该要点，或回答明显错误（得0分）
   - 💡 额外亮点：学生回答超出参考答案但正确合理（不计入分母）
5. 计算正确率：
   ```
   正确率 = 完整覆盖要点数 ÷ 参考要点总数 × 100%
   
   评级标准：
   ✅ 正确（优秀）：正确率 ≥ 90%
   ⚠️ 基本正确（良好）：70% ≤ 正确率 < 90%
   ❌ 错误（需改进）：正确率 < 70%
   ```
6. 生成改进建议

**重要**：必须完成所有学生的评估，不能只评估部分学生。

评估时参考：
- `references/提示词模板.md` - 获取分年级评估提示词
- `references/合理额外答案白名单.md` - 避免语义等价误判
- `references/手写识别纠错.md` - 处理OCR识别错误

### 步骤4：班级统计分析（批量≥10张时生成）

如学生数量≥10人，执行此步骤：

1. 正确率分布统计：
   - ✅ 正确（≥90%）：X人（XX%）
   - ⚠️ 基本正确（70-89%）：X人（XX%）
   - ❌ 需改进（<70%）：X人（XX%）

2. 平均正确率计算
3. 要点遗漏率统计
4. 班级整体水平评定

### 步骤5：错误原因诊断

分析全班集中错误的具体原因：

1. 收集所有错误样例
2. 归类错误类型（审题不清、知识欠缺、表达不当等）
3. 生成原因分析报告

参考 `references/错误原因诊断.md` 获取常见错误归类及原因分析。

### 步骤6：生成改进方案

针对每个薄弱要点，生成改进方案：

1. 设定改进目标（将遗漏率从XX%降至YY%）
2. 制定改进措施：
   - 课堂策略（3条具体建议）
   - 练习设计（3个具体练习）
   - 评价标准（明确打分标准）
3. 制定跟进计划（时间节点+检测方式）

参考 `references/改进方案库.md` 获取针对不同薄弱点的改进措施。

### 步骤7：生成完整报告文档

生成两种格式的报告：

#### 7.1 Markdown格式（用于预览）

使用 `write_to_file` 生成 Markdown 报告，包含：
- 逐生评估详情
- 班级整体学情统计
- 共性问题与错误原因分析
- 针对性改进方案
- 特别关注名单

保存位置：`[artifact_directory]/班级阅读答题学情分析报告.md`

#### 7.2 Word格式（用于打印和分享）

使用 `docx` 技能生成专业的Word文档，包含：
- 封面页（标题+日期+班级信息）
- 逐生评估详情（带格式）
- 统计表格（用Table实现）
- 问题分析（带项目符号）
- 改进方案（带复选框）
- 页眉页脚（页码+报告标题）

保存位置：`[artifact_directory]/班级阅读答题学情分析报告.docx`

### 步骤8：展示结果

使用 `open_result_view` 工具展示生成的报告。

**这是最后一步**：执行完这步，任务才算完成。

## 评估标准

### 正确率计算公式

```
正确率 = 完整覆盖要点数 ÷ 参考要点总数 × 100%

评级标准：
✅ 正确（优秀）：正确率 ≥ 90%
⚠️ 基本正确（良好）：70% ≤ 正确率 < 90%
❌ 错误（需改进）：正确率 < 70%
```

### 要点覆盖判定规则

| 判定 | 条件 | 得分 |
|------|------|------|
| ✅ 完整覆盖 | 学生回答完整、准确包含该要点的核心含义 | 1分 |
| ⚠️ 部分覆盖 | 学生回答涉及该要点，但表述不完整或部分错误 | 0.5分 |
| ❌ 缺失 | 学生回答未涉及该要点，或回答明显错误 | 0分 |
| 💡 额外亮点 | 学生回答超出参考答案但正确合理 | 不计入分母 |

### 语义等价判定

- 学生使用不同表达但意思正确 → 视为✅完整覆盖
- 参考 `references/合理额外答案白名单.md` 避免误判

### 分年级语言适配

根据年级使用适当的评价语言：
- **1-2年级**：使用简单词汇和短句，评价语气温和鼓励
- **3-4年级**：可使用较长句子，适当引入简单关联词
- **5-6年级**：语言可更加规范专业，可关联课标能力维度

参考 `references/分级表达资源.md` 获取各年级评价话术。

## 注意事项

1. **图片格式**：支持 PNG、JPG、JPEG
2. **批量上传**：单次最多50张，超过可分批
3. **正向评价**：即使❌错误，也用建设性语言给出改进方向
4. **数据溯源**：识别文本与原始图片关联存储
5. **错误归因**：不仅指出错误，更要分析原因并给出解决方案
6. **分年级语言**：根据年级使用适当的评价语言（参考 `references/分级表达资源.md`）

## 故障恢复指南

### 如中途被打断或停止：

**情况1：读完图片后停止了**
- 立即继续执行步骤3（逐人评估）
- 不要再次读取图片
- 直接基于已读取的图片内容继续分析

**情况2：分析部分学生后停止了**
- 检查已评估的学生数量
- 继续评估剩余学生
- 不要重复评估已完成的

**情况3：生成报告时出错**
- 记录错误信息
- 尝试用不同方式生成报告
- 如Markdown失败，直接生成Word
- 如Word失败，至少输出Markdown

**情况4：用户说"继续"或"接着做"**
- 不要再次读取图片
- 从中断的地方继续执行
- 完成所有剩余步骤

## 成功标准

必须满足以下所有条件，才能认为任务完成：

- ✅ 已读取所有学生作答图片
- ✅ 已评估所有学生的作答（逐人评估）
- ✅ 已生成班级统计数据（如≥10人）
- ✅ 已分析问题原因（如≥10人）
- ✅ 已生成改进方案（如≥10人）
- ✅ 已生成完整报告（Markdown + Word）
- ✅ 已使用 `open_result_view` 展示结果
- ✅ 没有遗漏任何步骤

**只有满足以上所有条件，才能停止。否则，继续执行。**

## 脚本使用

此技能包含以下可执行脚本：

### `scripts/evaluate_reading.py`

核心评估脚本，支持单张和批量图片评估。

**功能**：
- 图像预处理（倾斜校正、亮度调整等）
- OCR识别
- 错别字纠正
- 语义评估
- 批量处理（最多50张）

### `scripts/class_report.py`

班级学情分析报告生成脚本。

**功能**：
- 构建班级报告数据结构
- 生成整体概览
- 分析各要点掌握情况
- 生成教学建议

**注意**：脚本由技能自动调用，无需手动执行。
